background

Schneider National is a logistics giant, operating one of the largest freight networks in North America with over 9,000 trucks, 17,000+ daily users and a wide-spread service reach across the continent. Schneider FreightPower® is its digital platform that connects shippers and carriers for efficient load matching, booking, and tracking.

PROBLEM

Truckers & Brokers lack the ability to search for multiple freights and specify desired travel distances.

Schneider’s freight search platform originally allowed users to select criteria such as a single capacity type, origin, destination, and date ranges for pickup and drop- off. However, users had to scroll past top recommendations to access these search filters, which slowed down the search process. The platform also lacked features for selecting multiple capacity types, setting maximum miles to be traveled, and filtering loads by permissible truck weight, leading to limited customization and efficiency for drivers. These limitations restricted user flexibility, search accuracy, and accessibility, impacting overall operational effectiveness.

SOLUTION

Trucker Search Optimization: Addressing Search Limitations in Freight Tools

SOLUTION #1

Multi-capacity Load Selection

  • Users can now select multiple capacity types (e.g., Power-Only, Refrigerated, etc.) in a single search.
  • This allows freight managers and drivers to view a wider range of options and customize their search criteria based on various types of freight, leading to better matches.
  • Previously, only one capacity type could be selected, limiting search flexibility.
SOLUTION #2

Multi-capacity Load Selection

  • The platform now adapts has a mobile app design.
  • Independent drivers can now easily access search filters and results on phones.
  • This improves usability on mobile devices.
SOLUTION #3

Miles to Be Traveled Feature

  • Users can set a maximum travel distance for loads.
  • This ensures relevant results based on desired distances to be travelled.
  • There was no such feature before.
SOLUTION #4

Truck Weight Filtering

  • Users can filter loads by truck weight.
  • This ensures compliance with weight limits.
  • Previously, no weight filter was available.
SOLUTION #5

Above-the-Fold Search Interface

  • Search filters are now visible immediately.
  • Users can start searching without scrolling.
  • Previously, filters were hidden below the fold.
Current Stage in Enhancing Schneider's Freight Search Platform

At this stage, I’ve completed the development of the desktop version using ReactJS and am prototyping the mobile app to ensure a seamless user experience across platforms. I'm also actively conducting usability tests on the web version, gathering feedback to refine the design. Meanwhile, the team is focused on optimizing the search algorithm and integrating the PC Miler API, which will enhance route accuracy and improve search functionality, bringing us closer to a fully optimized freight search solution.
